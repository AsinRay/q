## debian 相关

### root 用户
在debian系统下，默认使用的是dash解释器，当登录login shell 时，会执行～/.profile文件中的内容，其中的脚本会查找存在~/.bashrc的文件，如果存在，就执行它。
我们看一下root用户下的~/.profile文件的内容：

```sh
# ~/.profile: executed by Bourne-compatible login shells.

if [ "$BASH" ]; then
  if [ -f ~/.bashrc ]; then
    . ~/.bashrc
  fi
fi

mesg n || true

```
其中的第一行已明确说明这个一个兼容Bourne shell的login shell

再看一下root用户默认的~/.bashrc

```sh
# ~/.bashrc: executed by bash(1) for non-login shells.

# Note: PS1 and umask are already set in /etc/profile. You should not
# need this unless you want different defaults for root.
# PS1='${debian_chroot:+($debian_chroot)}\h:\w\$ '
# umask 022

# You may uncomment the following lines if you want `ls' to be colorized:
# export LS_OPTIONS='--color=auto'
# eval "`dircolors`"
# alias ls='ls $LS_OPTIONS'
# alias ll='ls $LS_OPTIONS -l'
# alias l='ls $LS_OPTIONS -lA'
#
# Some more alias to avoid making mistakes:
# alias rm='rm -i'
# alias cp='cp -i'
# alias mv='mv -i'
PATH=/usr/local/sbin:/usr/sbin:/sbin:$PATH

```
~/.bashrc第一行的注释已明确地说明它是一个non-login shell. 但并不会被自动执行，仅仅是为了让～/.profile来调用的。

有时我们使用 su 命令切换到root用户，发现很多命令提示找不到;但是使用 su - 命令切换到root用户时
可以找到命令并执行，这是为什么呢？

### 非root用户

同样的存在文件~/.profile,其内容如下：

```sh
# ~/.profile: executed by the command interpreter for login shells.
# This file is not read by bash(1), if ~/.bash_profile or ~/.bash_login
# exists.
# see /usr/share/doc/bash/examples/startup-files for examples.
# the files are located in the bash-doc package.

# the default umask is set in /etc/profile; for setting the umask
# for ssh logins, install and configure the libpam-umask package.
#umask 022

# if running bash
if [ -n "$BASH_VERSION" ]; then
    # include .bashrc if it exists
    if [ -f "$HOME/.bashrc" ]; then
        . "$HOME/.bashrc"
    fi
fi

# set PATH so it includes user's private bin if it exists
if [ -d "$HOME/bin" ] ; then
    PATH="$HOME/bin:$PATH"
fi

# set PATH so it includes user's private bin if it exists
if [ -d "$HOME/.local/bin" ] ; then
    PATH="$HOME/.local/bin:$PATH"
fi

```
通过注释我们可以看到它同root用户下的.profile文件很相似,也是为了让login shell调用的，
区别在于当存在文件 ~/.bash_profile 或者 ~/.bash_login时，它不会被执行。
